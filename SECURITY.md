# Security Policy

## Reporting Security Vulnerabilities

Microsoft takes the security of our software and services seriously.

If you believe you have found a security vulnerability in this repository, please report it to the Microsoft Security Response Center (MSRC):

- **MSRC Reporting Portal:** https://msrc.microsoft.com/create-report
- **Microsoft Security Response Center:** https://www.microsoft.com/msrc

Please **do not** report security vulnerabilities through public GitHub issues, discussions, or pull requests.

## Supported Versions

Security updates for this repository are provided for the active development branch.

| Version | Supported |
| ------- | --------- |
| main (default branch) | :white_check_mark: |
| Older branches, tags, and snapshots | :x: |

## Repository Scope

This repository mainly contains training demos, templates, and experiment assets.

- Security issues in repository code/content are handled through this policy.
- Third-party products and dependencies referenced by demos may follow separate disclosure and patch policies from their maintainers.

## What to Include in a Report

To help us triage and validate quickly, include:

- A clear description of the issue and expected security impact
- Reproduction steps and prerequisites
- Affected paths/files and branch or commit SHA
- Proof-of-concept details, if available
- Suggested mitigation, if known
