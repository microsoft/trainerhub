# Learning-Day

Learning Day materials and resources from team learning sessions.

## Purpose

Store assets from CAIP Learning Day events that are execution-ready and reusable.

This includes demos, code samples, scripts, and practical guides that can be run or adapted in real environments.

## What Belongs Here

**Included:**

- Runnable demos and code samples from Learning Day sessions
- Scripts and automation tools shared during learning sessions
- Lab instructions and step-by-step guides
- Practical exercises with validation steps
- Quick-start templates used in learning sessions

**Not Included:**

- Presentation slides or recording links → Use SharePoint
- General knowledge articles without executable components → Use SharePoint
- Content that clearly belongs to a specific domain → Use domain folders (AI, Fabric, GitHub, Copilot)

## Migration Path

When Learning Day content becomes stable and domain-specific:

1. Review if the asset is clearly tied to one domain (AI, Fabric, GitHub, or Copilot)
2. If yes, move it to the appropriate domain folder's relevant category
3. Update any cross-references or documentation

If the content spans multiple domains, it can remain here.

## Contribution Guidelines

- Use English as the default language for new content
- Include clear prerequisites and setup instructions
- Ensure demos are reproducible in a clean environment
- Add a local README for each asset explaining objective, setup, and validation
- Tag assets with applicable domains in metadata when practical

## Examples

- Learning Day demo: RAG implementation with Azure AI Search and OpenAI
- Learning Day lab: GitHub Copilot workspace setup automation script
- Learning Day exercise: Building a Fabric notebook with data visualization
- Learning Day sample: Multi-domain cost optimization query script
