# Emerging-Technologies

Experimental content for technologies not yet in main domain folders.

## Purpose

Provide a sandbox for exploring new technologies, preview features, and experimental integrations before they become standard practice.

This folder supports early adoption and knowledge sharing while maintaining stability in main domain folders.

## What Belongs Here

**Included:**

- Preview and beta feature explorations
- New service or product evaluations
- Experimental integrations and proof-of-concepts
- Early-stage technology assessments
- Content for technologies being considered for domain addition

**Not Included:**

- Generally available (GA) features → Move to domain folders
- Deprecated previews that won't reach GA → Archive or delete
- Unrelated experiments without training value
- Personal projects without team benefit

## Lifecycle and Graduation

Content in Emerging-Technologies should be reviewed regularly to determine status:

### Status: Experimental

- Technology is in preview or early adoption
- Team is evaluating fit and value
- Content may change frequently
- Use in production not recommended

### Status: Maturing

- Technology is approaching GA or proven valuable
- Usage patterns are established
- Documentation is stabilizing
- Consider preparing for migration to domain folder

### Status: Graduate to Domain

- Technology is GA and widely used
- Clear fit with existing domain (AI, Fabric, GitHub, Copilot)
- Content is stable and reusable
- **Action**: Move to appropriate domain folder

### Status: Archive

- Technology preview was cancelled
- Team decided not to adopt
- Superseded by better alternative
- **Action**: Archive with context or delete

## Contribution Guidelines

- Use English as the default language for new content
- Clearly mark content as "Experimental" or "Preview"
- Document the technology's current status (preview, beta, GA)
- Include evaluation notes: benefits, limitations, use cases
- Add date of last review or update
- Note if content is outdated or superseded

## Review Cadence

Recommended review: Every 90 days

Questions to answer during review:

1. Is this technology now GA and ready for a domain folder?
2. Is it still relevant to team training goals?
3. Should it be archived or deleted?
4. Does it need updating based on recent changes?

## Examples

- Preview feature demo: Azure OpenAI with new model
- Experimental integration: GitHub Copilot with emerging tool
- Technology evaluation: New Fabric feature assessment
- Proof-of-concept: Cross-service experimental pattern
- Beta service exploration: New Azure service trial
